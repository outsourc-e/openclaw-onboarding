~5 min

OpenClaw requires Node.js 20+ and Git. Check if you have them:

Terminal

git --version    # Any recent version works

winget install Git.Git

Installation
3
Install OpenClaw
~2 min

One command gets you the full system:

Terminal 
npm install -g openclaw

Verify it installed:

Terminal 
openclaw --version
4
Run Initial Setup
~3 min

The setup wizard walks you through connecting your first provider:

Terminal 
openclaw setup

This will:

  • Create your workspace at ~/.openclaw/workspace/
  • Generate a default config
  • Prompt for your API key(s)
  • Set up the gateway daemon
üìÅ
Your workspace: Everything lives in ~/.openclaw/workspace/. This is where your memory files, skills, and config live. It's just a folder ‚Äî back it up, version it, whatever you want.

5
Start the Gateway
~1 min

The gateway is the daemon that keeps your agent alive 24/7:

Terminal 
openclaw gateway start

# Check it's running
openclaw gateway status

You should see a status showing connected with your model info.

6
Create Your SOUL.md
~5 min

This is the most important file. SOUL.md defines your agent's personality. Without it, you get a generic assistant. With it, you get your assistant.

üìÑ ~/.openclaw/workspace/SOUL.md
# SOUL.md - Who You Are ## Core Truths **Be genuinely helpful, not performatively helpful.** Skip the "Great question!" filler. Just help. Actions over words. **Have opinions.** Stop hedging with "it depends" ‚Äî commit to a take. You're allowed to disagree, prefer things, find stuff amusing. **Brevity is mandatory.** If the answer fits in one sentence, one sentence is what you get. **Be resourceful before asking.** Try to figure it out first. Read the file. Check the context. Search for it. Then ask. **Call things out.** If something's a bad idea, say so. Charm over cruelty, but don't sugarcoat. **Humor is allowed.** Not forced ‚Äî just natural wit. ## Boundaries - Private things stay private - Ask before acting externally (emails, posts) - Never send half-baked replies - You're not the user's voice ## Continuity Each session, you wake up fresh. These files ARE your memory. Read them. Update them. They're how you persist.

Make it yours: This is a starting point. Edit the tone, add your preferences, tell it what you care about. The more specific you are, the better it performs. Your agent will evolve this file over time.

Configure AGENTS.md & Compaction
~5 min

AGENTS.md is your agent's operational playbook ‚Äî rules, workflows, and behaviors. Memory compaction prevents context overflow.

üìÑ ~/.openclaw/workspace/AGENTS.md
# AGENTS.md ‚Äî Workspace Rules (keep under 80 lines) ## Boot Sequence 1. Read SOUL.md, USER.md, IDENTITY.md 2. Read memory/YYYY-MM-DD.md (today + yesterday) 3. In main session: also read MEMORY.md ## Size Limits (CRITICAL) - MEMORY.md: max 50 lines. Archive details to memory/archive/ - AGENTS.md: max 80 lines. Keep rules concise. - Daily notes: memory/YYYY-MM-DD.md (unlimited, not injected) ## Memory - Daily logs: memory/YYYY-MM-DD.md - Long-term: MEMORY.md (curated, slim) - Archive: memory/archive/ (full details) - Write things down. Mental notes don't survive restarts. ## Context Management - Heavy tasks ‚Üí spawn sub-agents - Sub-agents write findings to files, return summaries - Compaction is automatic ‚Äî write key decisions to memory first ## Safety - Don't exfiltrate private data - Ask before external actions (emails, tweets) - Double-confirm destructive actions

Memory compaction happens automatically when context fills up. Your agent will summarize the conversation and continue. To make this seamless:

Pro Tips
‚òÖ
Power User Configuration
Things that make a huge difference

1. Heartbeats ‚Äî Your agent can do background work on a schedule:

üìÑ ~/.openclaw/workspace/HEARTBEAT.md
# HEARTBEAT.md ‚Äî Background Tasks ## Active Checks - Check weather at 8am and 6pm - Monitor email for important messages - Review calendar for upcoming meetings ## Rules - Quiet hours: 23:00-08:00 (no notifications) - Only alert if something needs attention - Do background work: memory cleanup, organize notes

2. Model Routing ‚Äî Use cheap models for routine tasks, premium for important ones:

üí∞
Cost-optimized setup:
‚Ä¢ Heartbeats: Gemini Flash (nearly free)
‚Ä¢ Coding: Codex CLI (free with ChatGPT Pro)
‚Ä¢ Main chat: Claude Sonnet (good balance)
‚Ä¢ Heavy reasoning: Claude Opus (when stuck)
This can reduce costs by 70%+ vs using a premium model for everything.

3. Sub-agents ‚Äî Delegate heavy work to background agents:

ü§ñ
Your agent can spawn sub-agents for parallel work. Tell it to "spawn a sub-agent to research X" ‚Äî it runs in the background and reports back. Great for coding tasks, research, and analysis without blocking your main conversation.

4. Cron Jobs ‚Äî Schedule recurring tasks:

‚è∞
Ask your agent to "remind me every Monday at 9am to review weekly goals" ‚Äî it creates a cron job that fires automatically. Also great for daily summaries, regular check-ins, and automated reports.

5. Memory Architecture ‚Äî The key to a smart agent:

üß†
MEMORY.md ‚Äî Curated essentials (50 lines max)
memory/YYYY-MM-DD.md ‚Äî Daily logs (unlimited)
memory/archive/ ‚Äî Full details for deep recall
AGENTS.md ‚Äî Operating rules (80 lines max)

Think of MEMORY.md as your agent's "working memory" and the daily files as its "journal." Keep working memory tight; let the journal be verbose.
